= Schnellstart für die Migration zu Google Cloud VMware Engine mit dem VMware workloads migration advisor
:allow-uri-read: 
:icons: font
:imagesdir: ./media/


[role="lead"]
Beginnen Sie mit dem VMware Migration Advisor, um Ihre aktuelle Infrastruktur zu Google Cloud VMware Engine zu migrieren und dabei Google Cloud NetApp Volumes als externe NFS-Datenspeicher zu verwenden.

Bevor Sie beginnen, sollten Sie ein Verständnis für das https://docs.netapp.com/us-en/workload-setup-admin/permissions-reference.html["Berechtigungen für NetApp Workload Factory"^] haben.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-1.png["Eins"]Melden Sie sich bei Workload Factory an
[role="quick-margin-para"]
Sie müssen https://docs.netapp.com/us-en/workload-setup-admin/sign-up-saas.html["Einrichten eines Kontos bei Workload Factory"^] und melden Sie sich mit einem der https://docs.netapp.com/us-en/workload-setup-admin/console-experiences.html["Konsolenerfahrungen"^] .

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-2.png["Zwei"]Anmeldeinformationen und Berechtigungen hinzufügen
[role="quick-margin-para"]
Wählen Sie die https://docs.netapp.com/us-en/workload-setup-admin/permissions-reference.html["Berechtigungsrichtlinien"] um Ihren Bedürfnissen gerecht zu werden.

[role="quick-margin-para"]
Wenn Sie keine Berechtigungen erteilen, können Sie Workload Factory für VMware verwenden, um teilweise fertiggestellte Codebeispiele zu kopieren.

[role="quick-margin-para"]
Wenn Sie Berechtigungen erteilen, müssen Sie  https://docs.netapp.com/us-en/workload-setup-admin/add-credentials.html["Anmeldeinformationen manuell zu einem Konto hinzufügen"] bereitstellen, einschließlich eines GCP-Dienstkontos mit den erforderlichen Berechtigungen zum Anzeigen und Verwalten von Google Cloud NetApp Volumes-Ressourcen, zur VPC-Konfiguration und zur Google Cloud VMware Engine-Datenspeicheranbindung.

[role="quick-margin-para"]
https://docs.netapp.com/us-en/workload-setup-admin/add-credentials.html["Erfahren Sie, wie Sie Anmeldeinformationen und Berechtigungen hinzufügen"^].

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-3.png["Drittens"]Hochladen von VM-Inventardaten
[role="quick-margin-para"]
Sie können das Planungscenter verwenden, um Inventardetails zu Ihrer aktuellen VM-Umgebung hochzuladen.  Verwenden Sie das Datenerfassungsskript, RVTools oder NetApp Data Infrastructure Insights, um Ihre aktuelle VM-Konfiguration zu erfassen und die Daten in das Planungscenter hochzuladen.

[role="quick-margin-para"]
link:upload-vm-inventory.html["Erfahren Sie, wie Sie VM-Inventardaten hochladen"^] .

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-4.png["Vier"] Erstellen Sie mithilfe des VMware Migration Advisor einen Implementierungsplan
[role="quick-margin-para"]
Starten Sie den VMware Migration Advisor und konfigurieren Sie die VM-Umgebung, die Sie auf der Google Cloud VMware Engine-Infrastruktur unter Verwendung von Google Cloud NetApp Volumes als externe NFS-Datenspeicher erstellen möchten.

[role="quick-margin-para"]
link:launch-migration-advisor-gcve.html["Lesen Sie, wie Sie mit dem VMware Migrationsberater einen Bereitstellungsplan erstellen"].
